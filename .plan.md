# Architecture Delta: "Two Independent Agents" → "One Top-Level Agent + Two Internal Sessions"

## Current State (v2 as-implemented)

### What Exists and Works
- **Plugin layer**: hooks, feedback, overlay, patch queue, tools, merge, rollback — all solid (318 tests passing)
- **Patch Queue + State Machine**: PatchQueueManager with 8-state lifecycle, index.json, file locking
- **7 Agent Tools**: registered via `api.registerTool()`, working in OpenClaw runtime
- **ReviewOrchestrator**: coordinates review flow, has `trySpawnReviewAgent()` placeholder (always returns false → LLM fallback)
- **NotifyManager**: debounce, digest, dedupe, flood control
- **Config**: `reviewMode` 4 modes, `agents.review` / `agents.notify` nested config, backward compat

### What's Wrong (Architecture Mismatch)
1. `agents/skill-evolution-review.json` and `agents/skill-evolution-notify.json` are standalone JSON files — OpenClaw doesn't auto-discover them from plugin directories
2. OpenClaw agents come from `openclaw.json > agents.list` — plugin cannot register agents
3. `ReviewOrchestrator.trySpawnReviewAgent()` is a stub returning false — no session spawn API exists in the plugin API (`OpenClawPluginApi` has: `on`, `registerTool`, but NO `spawnAgent` / `createSession`)
4. Config uses `agents.review.agentId: 'skill-evolution-review'` referencing a non-existent top-level agent

## Target State

1. **One top-level agent** `skill-evolution` in `openclaw.json > agents.list` — visible in WebUI
2. **Review + Notify are internal sessions** managed by that agent, NOT separate top-level agents
3. **Plugin layer unchanged** — hooks, tools, queue, merge, rollback
4. **Config migrated**: `agents.review/notify` → `agent` (singular) + `sessions.review/notify`

### Phase B — Config Schema Migration
1. Types: `agents.review/notify` → `agent` (singular) + `sessions.review/notify`
   - `agent: { enabled: boolean; id: string; model: string | null }`
   - `sessions: { review: { enabled; reuse; thread; timeoutSeconds }; notify: { enabled; reuse; thread; timeoutSeconds } }`
2. Config defaults/validation: update `getDefaultConfig()`, `validateConfig()`, `deepMerge()`
3. Add `migrateAgentsCompat()`: if old `agents.review/notify` fields present, map to new structure
4. Update `openclaw.plugin.json` configSchema
5. Update config tests

### Phase C — ReviewOrchestrator Refactor
1. Remove `trySpawnReviewAgent()` (placeholder that always returns false)
2. Add `tryDelegateToAgentSession()`:
   - Check if `skill-evolution` agent exists in agents list (read `openclaw.json`)
   - If yes: use gateway HTTP API to send a task message to the agent
   - If no or fail: fall back to existing LLM review (current behavior preserved)
3. NotifyManager: same pattern — try to delegate notification to agent, fall back to structured log
4. Session lifecycle rules:
   - Review: created per-patch, reused within configurable window, timeout = `sessions.review.timeoutSeconds`
   - Notify: created per-digest-cycle or per-notification, timeout = `sessions.notify.timeoutSeconds`
   - Non-blocking: `session_end` hook never waits on agent session
5. Update orchestrator tests

### Phase D — Docs & Migration

2. Update README: single agent registration, migration from 2-agent to 1-agent
3. Update `examples/config.example.yaml`: new config structure
